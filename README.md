# Work Chat + AI Coach ユーザーテスト用プロトタイプ

## 1. プロトタイプの目的

このプロトタイプは、AI Coach が日々の業務中に右端の小さなサイドパネルとして常駐し、業務相談・記録・上司面談提案・3者面談・評価レポート作成まで支援する体験を、被評価者目線で検証するためのものです。

専用アプリを大きく開く体験ではなく、架空の業務チャット「Work Chat」で作業している横に AI Coach がいる状態を再現しています。ユーザーが入力した相談、頑張ったこと、困りごと、会議発言、合意事項は、AI返答・相談資料・会議ログ・評価レポートに反映されます。

## 2. 体験ステップ

1. 業務中にAI Coachを開く
2. 日常業務相談
3. 成果・困りごとの記録
4. 蓄積ダッシュボード
5. AIから上司面談の提案
6. 3者会議設定
7. 相談資料の自動作成
8. 3者面談シミュレーション
9. 面談後の整理
10. 評価レポート作成・上司承認
11. テストフィードバック確認

## 3. ローカルでの開き方

外部ライブラリ、本物のAI API、外部サービス連携は使っていません。

- `index.html` をブラウザで直接開く
- ローカルサーバーを使う場合は、リポジトリ直下で `python3 -m http.server 8000` を実行し、ブラウザで `http://localhost:8000` を開く

入力内容、テストメモ、リアクション、最後の回答はブラウザの `localStorage` に保存されます。初期状態に戻したい場合は、ブラウザの開発者ツールなどから該当ドメインの `localStorage` を削除してください。

## 4. ユーザーテストの進め方

- 参加者には細かく説明しすぎず、普段の業務画面の右端にAI Coachがいる前提で触ってもらう
- 操作しながら感じたことを声に出してもらう
- 各ステップのテストメモ欄に、違和感や期待を入力してもらう
- 最後のフィードバック画面で全体感を確認する
- 特に「邪魔ではないか」「業務中に使いたいか」「監視感がないか」を確認する
- 操作性の良し悪しよりも、AI Coachの価値や違和感に注目する

## 5. 確認したい観点

- 右端のサイドパネルとしてAI Coachがいる体験は自然か
- 業務中に邪魔ではなく、役に立ちそうに感じるか
- Work Chat風の作業画面と並行してAIに相談する体験にリアリティがあるか
- AIから上司面談を提案されることに納得感があるか
- 会議中にAIが補足・代弁することに安心感があるか
- 日々の相談・記録が評価レポートにつながる流れに違和感がないか
- 半期評価ではなく、必要なタイミングで成長相談する体験として成立しているか

## 6. 注意事項

- 実在人物名、社内固有名詞、実業務情報は使わない
- すべて架空のサンプルにする
- 実在サービスのロゴや商標は使わない
- 本物のAI APIや外部サービス連携は不要
- ユーザー入力を画面内に反映させることで、実際に使っている感覚を出す
- 説明ではなく、操作を通じて価値が伝わるプロトタイプにする
