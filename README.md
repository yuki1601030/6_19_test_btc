# AIコーチャーと3者面談

## 1. プロトタイプの目的

「AIコーチャーが日々の業務相談に伴走し、成果・困りごと・成長状況が蓄積されたタイミングで、上司との3者面談を提案・設定・進行・記録・評価レポート化まで支援する」という未来の働き方を、被評価者目線で体験するためのWebプロトタイプです。

本物のAI APIや社内システム連携は行わず、すべて架空のサンプルデータで構成しています。操作性の完成度よりも、ユーザーテストで世界観と価値仮説を確認することを目的にしています。

## 2. 体験ストーリー

ユーザーは架空の社員「佐藤さん」として体験します。佐藤さんは新サービス企画プロジェクトに参加しており、資料作成、会議参加、関係者調整、突発業務対応を日々行っています。

従来は半期評価のタイミングで半年分の成果や困りごとを思い出していました。このプロトタイプでは、AIコーチャーが日々の相談・成果・困りごとを蓄積し、一定期間データがたまったタイミングで上司との3者面談を提案します。

体験は以下の8ステップです。

1. 日常業務相談
2. 成果・困りごとの蓄積
3. AIから上司面談の提案
4. AIが3者会議を設定
5. AIが相談資料を自動作成
6. 3者面談シミュレーション
7. 相互理解と合意事項の整理
8. AIが評価レポートを作成し、上司が承認・加筆修正

## 3. ローカルでの開き方

ブラウザで `index.html` を直接開いてください。

```bash
open index.html
```

Windowsの場合は、エクスプローラーで `index.html` をダブルクリックして開けます。

簡易サーバーで確認したい場合は、以下を実行します。

```bash
python3 -m http.server 8000
```

その後、ブラウザで `http://localhost:8000` を開きます。

## 4. ユーザーテストで確認したいこと

- AIコーチャーとの日常相談に自然さを感じるか
- AIから上司面談を提案されることに価値を感じるか
- 3者面談でAIが補助することに安心感があるか
- 本人の困りごとをAIが代弁・補足する体験に期待できるか
- 上司との相互理解が深まるイメージがあるか
- 突発業務や遅れの背景が評価に反映されることに納得感があるか
- 最後の評価レポート作成・上司承認フローに違和感がないか
- 「半期評価」から「必要なタイミングでの成長相談」への変化が伝わるか

## 5. 注意事項

- 実在人物名、社内固有名詞、実業務情報は一切使っていません。
- すべて架空のサンプルとして作成しています。
- 本物のAI APIや外部サービス連携は不要な静的Webプロトタイプです。
- ユーザーテスト用なので、完成度よりも体験ストーリーが伝わることを優先しています。
- 画面上では見出し・カード・吹き出しを中心に整理し、3者面談シミュレーションと評価レポートは価値が伝わるよう具体的な内容を入れています。
