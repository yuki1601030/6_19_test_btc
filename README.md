# AI Coach 業務画面サイドパネルプロトタイプ

## 1. プロトタイプの目的

AI Coach が、普段の業務画面の右端に小さく常駐し、必要な時だけ相談・記録・面談提案・3者面談補助・評価レポート作成を支援する体験を検証するための、1ページ完結のユーザーテスト用 Web プロトタイプです。

専用アプリを開くのではなく、Work Chat、Work Calendar、Work Meeting、Work Record という架空の業務画面の横から AI Coach が出てくる状況を再現しています。ユーザー入力は AI 返答、相談メモ、3者面談ログ、評価レポートに反映され、`localStorage` に保存されます。

## 2. 体験ステップ

1. Work Chat で困りごとが発生する
2. 業務画面の横で記録する
3. 蓄積をサイドパネルで見る
4. Work Chat 上に面談提案が出る
5. Work Calendar 風に面談を設定する
6. 面談前に相談メモを作る
7. Work Meeting 中に AI Coach が補助する
8. Work Record でレポート化する

## 3. ローカルでの開き方

外部ライブラリ、本物の AI API、外部サービス連携は使っていません。

- `index.html` をブラウザで直接開く
- ローカルサーバーを使う場合は、リポジトリ直下で `python3 -m http.server 8000` を実行し、ブラウザで `http://localhost:8000` を開く

初期状態に戻したい場合は、ブラウザの開発者ツールなどから、このページの `localStorage` を削除してください。

## 4. ユーザーテストの進め方

- 参加者には細かく説明しすぎず、普段の業務画面の右端に AI Coach が出てくる前提で触ってもらう
- まず Work Chat 上で困りごとが発生している状況から始める
- 次に右端の AI Coach で相談、記録、面談提案、会議設定を体験してもらう
- 会議ステップでは、Work Meeting 画面の右端に AI Coach がいる状態で、本人発言と AI 補足を体験してもらう
- 操作中に感じたことを声に出してもらう
- 書記が発言・反応・違和感を記録する
- アプリ内にはテストメモ欄を置かない
- 特に「右端に出ることが自然か」「業務画面を邪魔しないか」「会議中の AI 補足が安心か怖いか」「レポート化に納得感があるか」を確認する

## 5. 確認したい観点

- Work Chat 中に AI Coach が右端から出てくる体験は自然か
- 業務中に邪魔ではなく使えそうか
- 入力や記録の負担は少ないか
- AI から面談提案されることに納得感があるか
- Work Meeting 中に AI が補足してくれることは安心か
- 本人が言いにくいことを AI が補足する体験は価値があるか
- 上司との相互理解が深まりそうか
- 最後のレポート化は便利か、怖いか
- 半期評価ではなく、必要な時に成長相談する体験として成立しているか

## 6. 注意事項

- 実在人物名、社内固有名詞、実業務情報は使わない
- 実在サービス名やロゴは使わない
- すべて架空のサンプルにする
- 本物の AI API や外部連携は不要
- 説明よりも、操作感と状況再現を優先する
- 情報量を増やしすぎない
- テストメモ機能は実装しない
