# AI Coach ユーザーテスト用プロトタイプ

## 1. プロトタイプの目的

このプロトタイプは、AIコーチャーが日々の業務相談に伴走し、成果・困りごと・成長状況が蓄積されたタイミングで、上司との3者面談を提案・設定・進行・記録・評価レポート化まで支援する体験を、被評価者目線で検証するためのものです。

説明資料ではなく、実際に触れる業務ツール風の画面として作成しています。ユーザー入力は画面内のAI返答、相談資料、会議ログ、評価レポートへ反映されます。

## 2. 体験ステップ

1. 日常業務相談
2. 成果・困りごとの記録
3. 2週間の蓄積ダッシュボード
4. AIから上司面談の提案
5. 3者会議設定
6. AIが相談資料を自動作成
7. 3者面談シミュレーション
8. 相互理解と合意事項
9. 評価レポート作成・上司承認
10. テストフィードバック確認

## 3. ローカルでの開き方

外部ライブラリや本物のAI APIは使っていません。以下のいずれかで確認できます。

- `index.html` をブラウザで直接開く
- ローカルサーバーを使う場合は、リポジトリ直下で `python3 -m http.server 8000` を実行し、ブラウザで `http://localhost:8000` を開く

入力内容、テストメモ、リアクション、最後の回答はブラウザの `localStorage` に保存されます。

## 4. ユーザーテストの進め方

- 参加者には細かく説明しすぎず、実際に画面を触ってもらう
- 操作しながら感じたことを声に出してもらう
- 各ステップのテストメモ欄に、違和感や期待を入力してもらう
- 最後のフィードバック画面で全体感を確認する
- 操作性の良し悪しよりも、AIコーチャーの価値や違和感に注目する

## 5. 確認したい観点

- AIコーチャーに日常の業務相談をしたいと思えるか
- 困りごとや遅れの背景をAIが記録・整理することに価値を感じるか
- AIから上司面談を提案されることに納得感があるか
- AIが3者面談で本人を補助・代弁することに安心感があるか
- 上司との相互理解が深まりそうか
- AIが評価レポートを作成し、上司が承認・加筆する流れに違和感がないか
- 半期評価ではなく、必要なタイミングで成長相談する体験として成立しているか

## 6. 注意事項

- 実在人物名、社内固有名詞、実業務情報は使わない
- すべて架空のサンプルにする
- 本物のAI APIや外部サービス連携は不要
- ユーザー入力を画面内に反映させることで、実際に使っている感覚を出す
- 説明ではなく、操作を通じて価値が伝わるプロトタイプにする
