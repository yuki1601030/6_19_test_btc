# AI Coach スケジュール遅れ相談プロトタイプ

## 1. プロトタイプの目的

AI Coach が、部下のスケジュール遅れ相談に伴走し、上司との面談設定・相談資料作成・会議中の補足説明まで支援する体験を検証するための、1ページ完結の Web プロトタイプです。

広い機能ではなく、以下の2つに絞っています。

- 部下が AI Coach にスケジュール遅れを相談すると、AI が上司との1時間後の面談を提案・設定し、相談資料も作る
- 会議中、上司が納得しない時、AI が1週間の予定・突発業務・後輩相談対応を可視化し、遅れの背景を説明する

## 2. 体験ステップ

1. Work Teams でスケジュール遅れを AI Coach に相談する
2. AI Coach が1時間後の上司面談と相談資料作成を提案する
3. Work Calendarで面談を確認し、Work Slides風の相談資料を自動作成する
4. 上司との会議が始まり、上司が遅れの原因を深掘りする
5. AI Coach が1週間のスケジュールを可視化し、突発業務・後輩相談対応を説明して上司が納得する

## 3. ローカルでの開き方

外部ライブラリ、本物の AI API、外部サービス連携は使っていません。

- `index.html` をブラウザで直接開く
- ローカルサーバーを使う場合は、リポジトリ直下で `python3 -m http.server 8000` を実行し、ブラウザで `http://localhost:8000` を開く

入力内容は `localStorage` に保存されます。初期状態に戻したい場合は、ブラウザの開発者ツールでこのページの `localStorage` を削除してください。

## 4. ユーザーテストの進め方

- 参加者には細かく説明しすぎず、M365系の業務画面で仕事をしている前提で触ってもらう
- Work Teams風画面でスケジュール遅れに困っている状況から始める
- 右端の AI Coach で相談し、面談提案、会議設定、相談資料作成まで体験してもらう
- Work Meeting風画面では、上司が納得しない場面を見てもらう
- その後、AI Coach が1週間のスケジュールを可視化して、突発業務や後輩相談対応を説明する体験を見てもらう
- 操作中に感じたことを声に出してもらう
- 書記が発言・反応・違和感を記録する

## 5. 確認したい観点

- M365系業務UIの中でAI Coachが右端に出る体験は自然か
- 業務中に邪魔ではなく使えそうか
- スケジュール遅れを AI Coach に相談したいと思えるか
- AI Coach から上司面談を提案されることに納得感があるか
- 1時間後の面談設定は自然か
- 相談資料の自動作成は便利か
- 会議中に AI が補足することは安心か、怖いか
- 週間スケジュール可視化によって遅れの背景が伝わるか
- 空白時間の実態が見えることに価値を感じるか
- 上司が納得する流れにリアリティがあるか
- 半期評価よりも、必要なタイミングで成長相談する体験として成立しているか

## 6. 注意事項

- 実在人物名、社内固有名詞、実業務情報は使わない
- 実在サービス名やロゴは使わない
- すべて架空のサンプルにする
- 本物の AI API や外部連携は不要
- 説明よりも、操作感と状況再現を優先する
- 情報量を増やしすぎない
- テストメモ機能は実装しない
